# دستور اجرایی جامع

## برنامه پذیرش آزمایشگاه (Laboratory Admission System)

این سند به‌عنوان **Specification + Execution Plan** برای پیاده‌سازی یک برنامه کامل پذیرش آزمایشگاه در GitHub تهیه شده است؛ به‌طوری‌که هم نیازهای عملیاتی را پوشش دهد و هم به یک برنامه اجرایی، قابل استقرار و قابل توسعه منجر شود.

---

## 1. هدف پروژه

ایجاد یک سامانه یکپارچه برای مدیریت پذیرش آزمایشگاه شامل:

* ثبت و مدیریت مراجعین/بیماران
* مدیریت آزمایش‌ها و نسخه‌ها
* زمان‌بندی و نوبت‌دهی
* پذیرش، نمونه‌گیری، و تحویل نتایج
* گزارش‌گیری مدیریتی

---

## 2. ذی‌نفعان (Stakeholders)

* پذیرش/منشی آزمایشگاه
* تکنسین آزمایشگاه
* پزشک / ارجاع‌دهنده
* مدیر آزمایشگاه
* بیمار / مراجعه‌کننده

---

## 3. قابلیت‌های اصلی (Core Features)

### 3.1 مدیریت کاربران و نقش‌ها

* نقش‌ها: Admin، پذیرش، تکنسین، پزشک، بیمار
* احراز هویت (Login / Logout)
* سطح دسترسی مبتنی بر نقش (RBAC)

### 3.2 پذیرش بیمار

* ثبت اطلاعات بیمار (نام، کد ملی، تاریخ تولد، تماس)
* جستجوی بیمار قبلی
* ایجاد پرونده پذیرش جدید

### 3.3 مدیریت آزمایش‌ها

* تعریف آزمایش (کد، نام، هزینه، زمان آماده‌سازی)
* دسته‌بندی آزمایش‌ها
* فعال/غیرفعال کردن آزمایش‌ها

### 3.4 ثبت نسخه و درخواست آزمایش

* ثبت نسخه پزشک
* انتخاب چند آزمایش برای هر پذیرش
* محاسبه خودکار هزینه

### 3.5 نوبت‌دهی و زمان‌بندی

* تعیین تاریخ و ساعت مراجعه
* محدودیت ظرفیت روزانه
* نمایش تقویم کاری

### 3.6 نمونه‌گیری و وضعیت آزمایش

* تغییر وضعیت پذیرش:

  * ثبت‌شده
  * نمونه‌گیری‌شده
  * در حال بررسی
  * آماده تحویل
* ثبت توضیحات تکنسین

### 3.7 نتایج آزمایش

* ثبت نتیجه (عددی/متنی)
* آپلود فایل (PDF)
* تاریخ و امضای مسئول فنی

### 3.8 گزارش‌گیری

* گزارش پذیرش روزانه/ماهانه
* گزارش درآمد
* گزارش آزمایش‌های پرتکرار

---

## 4. نیازمندی‌های غیرعملکردی (Non‑Functional)

* زبان: فارسی (RTL)
* امنیت: رمزنگاری رمز عبور (bcrypt / argon2)
* لاگ‌گیری عملیات مهم
* مقیاس‌پذیری (REST API)
* تست‌پذیری

## 8. معیار پذیرش (Acceptance Criteria)

* کاربر پذیرش بتواند بیمار جدید ثبت کند
* سیستم هزینه را خودکار محاسبه کند
* نتیجه آزمایش قابل دانلود باشد
* گزارش روزانه در کمتر از 3 ثانیه تولید شود
